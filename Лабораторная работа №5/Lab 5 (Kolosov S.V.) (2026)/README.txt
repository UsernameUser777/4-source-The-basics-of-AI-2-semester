ЛАБОРАТОРНАЯ РАБОТА №5 ПО КУРСУ "ОСНОВЫ ИИ"
============================================

Тема: Решение прямой задачи линейного программирования симплексным методом

Вариант №1: Оптимизация загрузки грузопассажирского судна
---------------------------------------------------------

Цель работы:
------------
Разработать программное обеспечение на языке Python для решения задачи линейного
программирования (ЛП) симплекс-методом, применительно к задаче оптимизации загрузки
грузопассажирского судна с целью максимизации прибыли. Программа должна не только
находить оптимальный план распределения грузов, но и проводить анализ устойчивости
(чувствительности) решения, включая расчёт теневых цен, допустимых изменений
ресурсов и минимально допустимых цен для невыгодных грузов, а также анализировать
различные сценарии изменения параметров задачи.

Описание задачи (вариант №1):
------------------------------
- Судно имеет 3 грузовых отсека:
  - Отсек 1: объём 500 м³, макс. грузоподъёмность 700 т
  - Отсек 2: объём 1000 м³, макс. грузоподъёмность 800 т
  - Отсек 3: объём 1500 м³, макс. грузоподъёмность 1300 т
- Перевозятся 5 типов грузов:
  - Груз 1: Мини-тракторы (0.35 т, 3 м³, 8 ден.ед./ед., 100 ед. доступно)
  - Груз 2: Бумага (1.6 т, 1 м³, 21.5 ден.ед./ед., 1000 ед. доступно)
  - Груз 3: Контейнеры (5 т, 6.5 м³, 51 ден.ед./ед., 200 ед. доступно)
  - Груз 4: Металлопрокат (35 т, 6 м³, 275 ден.ед./ед., 200 ед. доступно)
  - Груз 5: Пиломатериалы (4 т, 6 м³, 110 ден.ед./ед., 350 ед. доступно)

Целевая функция:
F = 8*(x11+x12+x13) + 21.5*(x21+x22+x23) + 51*(x31+x32+x33) +
    275*(x41+x42+x43) + 110*(x51+x52+x53) → max
где xij — количество единиц груза типа i в отсеке j.

Ограничения:
- По весу каждого отсека.
- По объёму каждого отсека.
- По наличию каждого типа груза.
- Неотрицательность переменных.

Состав проекта:
---------------
- main.py                 - основное приложение с графическим интерфейсом (tkinter)
- simplex_solver.py       - ядро симплекс-метода
- sensitivity_analysis.py - анализ устойчивости решения
- requirements.txt        - список зависимостей
- README.txt              - настоящий файл

Технологии:
-----------
- Python 3.13
- tkinter (графический интерфейс)
- numpy (численные вычисления)
- matplotlib (визуализация)

Установка зависимостей:
-----------------------
pip install -r requirements.txt

Запуск программы:
-----------------
python main.py

Особенности реализации:
-----------------------
- Полная поддержка кодировки UTF-8 для корректной работы с кириллицей
  (вывод отчётов, интерфейс, экспортируемые файлы).
  (Примечание: Изначально планировалась Windows-1251, но код был адаптирован под UTF-8
   для лучшей совместимости и современных стандартов).
- Детализированный вывод всех итераций симплекс-таблицы с разрешающими элементами.
- Автоматический расчёт теневых цен (двойственных оценок) и допустимых изменений
  ресурсов (верхние и нижние границы).
- Определение невыгодных грузов (с нулевым количеством в оптимальном плане) и
  расчёт минимально допустимой цены для их рентабельности.
- Анализ сценариев (изменение прибыли при изменении наличия грузов):
  - Пиломатериалы: 400 ед. (вместо 350 ед.)
  - Бумага: 900 ед. (вместо 1000 ед.)
  - Контейнеры: 100 ед. (вместо 200 ед.)
  - Анализ производится как приблизительно (через теневые цены), так и точно
    (путём повторного решения задачи с изменёнными ограничениями).
- Визуализация результатов в виде столбчатой диаграммы распределения грузов по типам.
- Возможность сохранения графиков в файл (PNG).
- Прогресс-бар для отслеживания состояния длительных операций (реализован в интерфейсе).
- Экспорт результатов в CSV и JSON форматы с подробной структурой данных и поддержкой UTF-8.
- Подробные комментарии ко всем функциям и методам на русском языке.
- 7 вкладок интерфейса: Исходные данные, Решение (встроенный), Анализ устойчивости (встроенный),
  Сценарии (встроенный), Итоговые результаты (встроенный), Графики (встроенный), Ручной ввод.
- Встроенный валидатор ввода данных в режиме "Ручной ввод" для предотвращения ошибок.
- Функция "Очистить результаты" для сброса состояния программы.
- Логирование событий и ошибок в файл app_log.log.
- Исправлены синтаксические ошибки и ошибки форматирования, приводившие к сбоям.
- Исправлена логика сохранения итераций симплекс-таблицы для корректного отображения оптимального решения.
- Исправлена ошибка в методе calculate_min_price_for_unprofitable_cargos в sensitivity_analysis.py.
- Исправлена ошибка в методе generate_stability_report, связанная с форматированием строк вместо чисел.
- Исправлена ошибка NameError для 'logger' в методе analyze_scenario в sensitivity_analysis.py.

Функция ручного ввода:
----------------------
- Пользователь может ввести свои собственные параметры отсеков (объём, вес) и грузов (название, вес, объём, цена, наличие).
- Программа решает задачу оптимизации для введённых параметров.
- Результаты ручного расчёта можно сравнить с встроенным вариантом.
- Функции анализа устойчивости, сценариев, графиков и экспорта доступны для обоих режимов.
